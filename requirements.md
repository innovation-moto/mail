# メールアプリ 要件定義書

**バージョン**: 1.0  
**作成日**: 2026-05-12  
**対象プロダクト**: AI搭載パーソナルメールクライアント

---

## 1. プロジェクト概要・ゴール

### 概要

IMAP/SMTPに対応したメールクライアントアプリ。Gmail・Outlook・独自ドメインなど任意のメールアカウントを接続できる。AI機能（返信文生成・要約・自動分類・スマート検索）を標準搭載し、メール処理の効率化を実現する。

### プロダクトゴール

- メールの読み書きを快適に行えるシンプルで高速なクライアントを提供する
- AIにより「読む・書く・探す・整理する」の4つの負担を大幅に軽減する
- デスクトップ（PC）で先行リリースし、その後モバイル（iOS/Android）へ展開する

---

## 2. 対象ユーザー・ユースケース

### 対象ユーザー

個人ユーザー（主に日本語話者）。複数のメールアカウントを持ち、日常的にメールで連絡・タスク管理を行う人。

### 主要ユースケース

| # | ユースケース | 概要 |
|---|---|---|
| UC-01 | メール受信・閲覧 | 受信トレイ一覧を確認し、メール本文を読む |
| UC-02 | メール作成・送信 | 新規メールを作成して送信する |
| UC-03 | 返信・転送 | 受信メールに返信・転送する |
| UC-04 | AI返信文生成 | AIが返信の下書きを自動生成し、編集して送信する |
| UC-05 | メール要約 | 長いメールスレッドをAIが要約して表示する |
| UC-06 | 自動分類 | AIが受信メールを重要度・カテゴリで自動分類する |
| UC-07 | スマート検索 | 自然言語でメールを検索・質問する |
| UC-08 | アカウント管理 | 複数メールアカウントの追加・切替・削除を行う |

---

## 3. プラットフォーム計画

### Phase 1：デスクトップアプリ（PCアプリ）

- **対応OS**: macOS / Windows
- **技術**: Electron + React（またはTauri + React）
- **配布方法**: ローカルインストール（.dmg / .exe）

### Phase 2：モバイルアプリ（スマホアプリ）

- **対応OS**: iOS / Android
- **技術**: React Native（Expo）
- **方針**: Phase 1のUI/ロジックを最大限コンポーネント共通化する

### クロスプラットフォーム設計方針

- ビジネスロジック（API通信・メール処理・AI連携）は共通のTypeScriptモジュールとして切り出す
- UIコンポーネントは各プラットフォーム向けに実装しつつ、デザインシステムを統一する
- バックエンド（AI処理・IMAP接続）はローカル or 軽量サーバーで対応

## 4. 機能要件

### 4-1. 基本機能

#### メール受信・表示

- [ ] IMAP接続によるメール受信（SSL/TLS対応）
- [ ] 受信トレイ・送信済み・ゴミ箱・フォルダ一覧の表示
- [ ] メール一覧表示（差出人・件名・日時・既読/未読）
- [ ] メール本文表示（テキスト / HTML両対応）
- [ ] 添付ファイルの表示・ダウンロード
- [ ] 既読/未読の切替
- [ ] スター・フラグ付け

#### メール作成・送信

- [ ] SMTP経由での送信（SSL/TLS対応）
- [ ] テキスト・リッチテキスト（HTML）でのメール作成
- [ ] ファイル添付（複数対応）
- [ ] 宛先補完（アドレス帳 / 過去の送受信履歴から）
- [ ] 下書き保存
- [ ] 返信・全員に返信・転送

#### アカウント管理

- [ ] 複数アカウントの登録（IMAP/SMTPの設定）
- [ ] アカウント切替
- [ ] パスワード等の設定変更・削除
- [ ] Gmailなど主要プロバイダのクイック設定テンプレート

#### メール整理

- [ ] フォルダ/ラベルへの手動移動
- [ ] アーカイブ・削除・ゴミ箱
- [ ] 検索（件名・差出人・本文・期間による絞り込み）
- [ ] スレッド表示（メールチェーンをまとめて表示）

#### 新着通知

- [ ] 新着メール受信時にOSネイティブ通知を表示（macOS / Windows対応）
- [ ] 通知クリックで該当メールを即座に開く
- [ ] 通知のON/OFFをアカウント・フォルダ単位で設定可能
- [ ] 重要度が「高」のメールのみ通知するフィルター設定
- [ ] アプリがバックグラウンド時でも通知を受け取れる（Electronのバックグラウンド動作）
- [ ] 通知の表示時間・サウンドのカスタマイズ

#### ブロック機能

- [ ] 特定の送信者アドレスをブロックリストに追加
- [ ] ブロックしたアドレスからのメールを自動的にゴミ箱へ移動（または既読スキップ）
- [ ] ブロックリストの一覧表示・管理画面
- [ ] ブロックリストへの追加はメール右クリックメニューから1クリックで可能
- [ ] ドメイン単位でのブロック（例：`@spam-domain.com` をまとめてブロック）

### 4-2. AI機能

#### AI-01：返信文の自動生成

| 項目 | 内容 |
|---|---|
| 概要 | 受信メールの内容を解析し、返信文の下書きをAIが自動生成する |
| トリガー | 返信ボタン押下時に「AIで下書きを生成」ボタンを表示 |
| 入力 | メール本文・件名・スレッド履歴 |
| 出力 | 返信文（トーン選択：丁寧・カジュアル・簡潔） |
| ユーザー操作 | 生成された文章を編集・承認して送信 |
| 注意点 | 生成はあくまで下書き。ユーザーが必ず確認・送信する |

#### AI-02：メールの要約

| 項目 | 内容 |
|---|---|
| 概要 | 長いメール本文やスレッドを読みやすく要約する |
| トリガー | メール表示時に「要約を見る」ボタン表示（本文が一定量以上の場合） |
| 入力 | メール本文 / スレッド全体 |
| 出力 | 3〜5行の日本語要約 + 要アクション事項の抽出 |

#### AI-03：自動分類・優先度付け

| 項目 | 内容 |
|---|---|
| 概要 | 受信メールをAIが自動でカテゴリ分類し、重要度スコアを付ける |
| カテゴリ例 | 重要・タスク・情報・ニュースレター・プロモーション・その他 |
| 優先度 | 高・中・低の3段階 |
| トリガー | 新着メール受信時に自動実行 |
| ユーザー操作 | 誤分類の場合はユーザーが手動で修正可能（学習に反映） |

#### AI-04：スマート検索・質問応答

| 項目 | 内容 |
|---|---|
| 概要 | 自然言語でメールに関して検索・質問できる |
| 入力例 | 「先月の山田さんとのやり取りを教えて」「〇〇の請求書はどこ？」 |
| 出力 | 関連メール一覧 + 質問への直接回答 |
| 技術 | ローカルベクトル検索（埋め込み） or API経由での検索拡張生成（RAG） |

## 5. 非機能要件

### パフォーマンス

- 受信トレイの初期表示：3秒以内
- メール本文表示：1秒以内
- AI生成のレスポンス：5秒以内（ローカルAI利用時は10秒以内）
- バックグラウンド同期：30秒ごとに自動チェック

### セキュリティ

- IMAP/SMTPはSSL/TLS必須
- メールアカウントのパスワード・APIキーはOSのKeychain / Credential Managerに保存（平文保存禁止）
- AI処理にメール本文を外部APIに送信する際はユーザーへの明示的な同意を取得
- AIに送信するデータは最小限に（本文のみ、個人情報は可能な限りマスク）

### プライバシー

- メールデータはローカルに保存（クラウド同期機能はPhase 2以降で検討）
- AI API連携はオプトイン方式（初期設定でOFF）

### ユーザビリティ

- 初回設定（アカウント登録）を5分以内に完了できること
- キーボードショートカット対応（PC版）
- ダークモード / ライトモード対応
- 日本語・英語のUI対応

### 可用性・拡張性

- オフライン時でも保存済みメールは閲覧可能
- プラグイン/拡張機能の追加を将来的に考慮した設計

## 6. 技術スタック案

### デスクトップアプリ構成（Electron + Next.js）

```
┌─────────────────────────────┐
│  Electron シェル             │
│  ┌───────────────────────┐  │
│  │  Next.js（UIレイヤー）  │  │
│  │  Tailwind CSS         │  │
│  │  shadcn/ui            │  │
│  └──────────┬────────────┘  │
│             │ IPC通信         │
│  ┌──────────▼────────────┐  │
│  │  Node.js（メインプロセス） │  │
│  │  IMAP/SMTP・SQLite     │  │
│  │  AI API連携            │  │
│  └───────────────────────┘  │
└─────────────────────────────┘
```

| 項目 | 技術 |
|---|---|
| デスクトップシェル | Electron |
| UIフレームワーク | Next.js（App Router）+ TypeScript |
| UIライブラリ | Tailwind CSS + shadcn/ui |
| 状態管理 | Zustand |
| プロセス間通信 | Electron IPC（contextBridge） |
| メール接続 | imapflow（IMAP）、nodemailer（SMTP） |
| ローカルDB | SQLite（better-sqlite3） |
| AI連携 | Google Gemini 1.5 Flash API（無料枠：1日100万トークン） |
| ベクトル検索 | sqlite-vss または LanceDB（スマート検索用） |

### モバイル版（Phase 2）

| 項目 | 選択肢 |
|---|---|
| フレームワーク | React Native（Expo） |
| 共通ロジック | PC版のビジネスロジック（TypeScript）を共有 |
